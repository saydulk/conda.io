Buttons #

Buttons are a key means of interacting with any application, whether for downloading the latest version or confirming that you want to continue. Below are the various buttons that are available.

You should use the default button style for all buttons that are not a primary action. Buttons that denote a positive action (creation, save, confirmation, and so on) should use the success class while buttons that denote a destructive action (trash, removal, and so on) should use alert.

Colors #

Consistent color use plays an important role in guiding the user's actions to ensure they're able to successfully navigate your site or application.

When in doubt, use default.

You should use the default button color for most actions. Where two or more actions are available, style the primary action with the default behavior, and add the secondary class to all other actions.

You should only use success for final actions that are positive such as finishing an account creation, signing up for a mailing list, or downloading software. The user should not have to click another button or link to finish an action after clicking on a success button.

Likewise, you should only use alert for actions that are considered destructive such as deleting an account.
